# Task T-906A — Auth runtime wiring
**Section:** S09 — Auth & Users / wiring prerequisite

## Purpose
Wire the auth runtime dependencies needed by T-906 through the approved application composition path, so auth service/dependency code can retrieve already-wired `PasswordHasher` and `TokenSigner` instances without constructing adapters directly.

## Depends on
- T-903
- T-904
- T-905
- T-905A
- T-505

## Allowed files
- `app/core/container.py`
- `app/core/wiring/auth.py`
- `app/core/wiring/__init__.py`
- `app/core/di.py`
- `app/core/lifespan.py`
- `app/core/wiring/tests/test_wiring_auth.py`
- `app/main/tests/test_app_factory.py`
- `tests/integration/test_lifespan.py`
- `docs/implementation/tasks/T-906.md`
- `docs/implementation/roadmap.md`

*Note: `app/core/di.py` may be edited only if a session provider or container retrieval helper needed by auth deps does not already exist. `app/core/lifespan.py` may be edited only to populate auth runtime dependencies in the container and preserve readiness ordering. `app/main/tests/test_app_factory.py` and `tests/integration/test_lifespan.py` may be edited only if container/lifespan behavior changes require test updates. `docs/implementation/tasks/T-906.md` and `docs/implementation/roadmap.md` may be edited only to add T-906A as a dependency or mark status.*

## Forbidden changes
- Do not implement login, refresh, logout, or current-user behavior.
- Do not implement auth service business logic.
- Do not implement `get_current_user` behavior beyond dependency retrieval scaffolding if needed.
- Do not modify `app/auth/persistence.py`.
- Do not add SQLAlchemy queries or raw SQL.
- Do not add DB migrations.
- Do not import `app.infrastructure.*` outside `app.core.wiring.*`.
- Do not import provider SDKs like `argon2` or `jwt` in wiring.
- Do not construct adapters in `app/auth/deps.py` or `app/api` modules.
- Do not add global `app.ports`.
- Do not introduce `use_cases/`.
- Do not modify `.importlinter`.
- Do not add `auth_service` to `Container` unless the task explicitly justifies singleton-vs-request lifecycle.

## Implementation requirements
- Create `app/core/wiring/auth.py`.
- Add setup functions:
    - `setup_password_hasher(settings: AppSettings) -> PasswordHasher`
    - `setup_token_signer(settings: AppSettings) -> TokenSigner`
- Wiring may import adapter classes from `app.auth.adapters.*`.
- Wiring must pass minimal settings/config values to adapters.
- Wiring must not import `argon2` or `JWT` provider SDK modules directly.
- Extend `Container` with typed Protocol fields:
    - `password_hasher: PasswordHasher | None`
    - `token_signer: TokenSigner | None`
- Register `password_hasher` and `token_signer` in lifespan startup before `app.state.ready = True`.
- Preserve shutdown/readiness ordering.
- Do not create/close external resources in this task unless a real lifecycle-managed auth resource is introduced.

## Tests required
- `setup_password_hasher(settings)` returns an object structurally satisfying `PasswordHasher`.
- `setup_token_signer(settings)` returns an object structurally satisfying `TokenSigner`.
- `Container` fields are populated after lifespan startup.
- Existing app factory/lifespan tests remain isolated from live services.
- No provider SDK imports appear in wiring.
- No direct infrastructure imports appear outside `app.core.wiring.*`.

## Acceptance criteria
- `app/core/wiring/auth.py` exists.
- `Container` has Protocol-typed `password_hasher` and `token_signer` fields.
- Lifespan populates those fields before readiness flips true.
- Auth adapters are constructed only through wiring.
- No provider SDK imports in wiring.
- No auth service business logic is implemented.
- T-906 dependency is patched to include T-906A.
- `make lint`, `make typecheck`, `make test` pass.

## Commands
- `make lint`
- `make typecheck`
- `make test`
- `uv run lint-imports`
- `uv run pytest app/core/wiring/tests/test_wiring_auth.py tests/integration/test_lifespan.py`

## Common failure modes
- Importing `argon2` or `jwt` in wiring.
- Adding fields typed with the adapter class instead of the Protocol.
- Flipping `app.state.ready = True` before wiring completes.
- Constructing adapters inside `deps.py` or `api.py`.

## Review checklist
- Container fields are Protocol-typed.
- No SDK imports in wiring.
- Readiness ordering unchanged.
- No business logic in wiring.
