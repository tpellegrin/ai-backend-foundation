## Purpose

Wire the auth runtime dependencies needed by T-906 through the approved application composition path, so auth service/dependency code can retrieve already-wired `PasswordHasher` and `TokenSigner` instances without constructing adapters directly.

This task also updates the import-linter contract only as needed to authorize the narrow composition-root edge from `app.core.wiring.auth` to concrete auth adapters in `app.auth.adapters`.

## Forbidden changes

- Do not implement login, refresh, logout, or current-user behavior.
- Do not implement auth service business logic.
- Do not implement `get_current_user` behavior beyond dependency retrieval scaffolding if needed.
- Do not modify `app/auth/persistence.py`.
- Do not add SQLAlchemy queries or raw SQL.
- Do not add DB migrations.
- Do not import `app.infrastructure.*` outside `app.core.wiring.*`.
- Do not import provider SDKs like `argon2` or `jwt` in wiring.
- Do not construct adapters in `app/auth/deps.py` or `app/api` modules.
- Do not add global `app.ports`.
- Do not introduce `use_cases/`.
- Do not weaken import-linter boundaries.
- Do not add broad wildcard import-linter exceptions.
- Do not allow non-wiring modules to import `app.auth.adapters`.
- Do not add `auth_service` to `Container` unless the task explicitly justifies singleton-vs-request lifecycle.

---

## Implementation requirements

- Create `app/core/wiring/auth.py`.
- Add setup functions:
    - `setup_password_hasher(settings: AppSettings) -> PasswordHasher`
    - `setup_token_signer(settings: AppSettings) -> TokenSigner`
- Wiring may import adapter classes from `app.auth.adapters.*`.
- Wiring must pass minimal settings/config values to adapters.
- Wiring must not import `argon2` or JWT provider SDK modules directly.
- Extend `Container` with typed `Protocol` fields:
    - `password_hasher: PasswordHasher | None`
    - `token_signer: TokenSigner | None`
- Register `password_hasher` and `token_signer` in lifespan startup before `app.state.ready = True`.
- Preserve shutdown/readiness ordering.
- Do not create/close external resources in this task unless a real lifecycle-managed auth resource is introduced.

---

## Import-linter contract update

If `uv run lint-imports` fails because `app.core.wiring.auth` imports `app.auth.adapters`, update `.importlinter` with the narrowest allowed edge that reflects the accepted composition-root rule.

The intended rule is:

- `app.core.wiring.auth` may import concrete auth adapters from `app.auth.adapters`.
- `app.auth.service`, `app.auth.deps`, `app.api`, and other non-wiring modules must not import concrete auth adapters.
- No broad exception should allow arbitrary `app.** -> app.auth.adapters.**` imports.

Do not use dynamic imports, local imports, re-export shims, or wildcard ignore rules to hide the edge.

## Common failure modes

- Importing `argon2` or `jwt` in wiring.
- Adding fields typed with the adapter class instead of the `Protocol`.
- Flipping `app.state.ready = True` before wiring completes.
- Constructing adapters inside `deps.py` or `api.py`.
- Leaving `uv run lint-imports` failing after adding the legitimate wiring edge.
- Adding a broad wildcard import-linter exception.
- Allowing `app.auth.service`, `app.auth.deps`, or `app.api` to import `app.auth.adapters`.
- Using dynamic imports to avoid import-linter instead of updating the contract explicitly.
