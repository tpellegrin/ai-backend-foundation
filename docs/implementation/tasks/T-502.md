Purpose: ship two pure, framework-thin utilities — a security-headers ASGI middleware class and a cursor pagination FastAPI dependency. **Framework registration is out of scope**: this task does not mount, install, or wire either symbol onto any FastAPI instance, and it does not touch CORS. Registration is owned by T-505.

Depends on: T-203, T-301.

Allowed files: `app/api/__init__.py`, `app/api/security_headers.py`, `app/api/pagination.py`, `app/api/tests/test_security_headers.py`, `app/api/tests/test_pagination.py`.

Forbidden

- Do **not** register the middleware on any `FastAPI` / `Starlette` instance in this task (no `app.add_middleware(...)`, no `create_app()` edits).
- Do **not** configure or register CORS here. `Api` settings (from T-301) are read by T-505, which registers `starlette.middleware.cors.CORSMiddleware`.
- Do **not** import `app.core.app_factory` or reference `app.state`.
- Do **not** widen scope to any other `app/api/*` file.

Implementation requirements

- `SecurityHeadersMiddleware` is a pure ASGI middleware class (BaseHTTPMiddleware or ASGI3 callable) that sets every one of the following headers on every outbound response:
  - `Strict-Transport-Security: max-age=63072000; includeSubDomains`
  - `X-Content-Type-Options: nosniff`
  - `Referrer-Policy: no-referrer`
  - `X-Frame-Options: DENY`
  - `Content-Security-Policy: default-src 'none'`
- Header emission may be gated by the `Api.security_headers_enabled` boolean passed at middleware construction time (default `True`). Middleware must not read settings directly — the caller passes the flag.
- `pagination.py` exposes a `CursorParams` value object plus a FastAPI dependency producing a `CursorParams` from the query string (opaque `cursor: str | None`, `limit: int` with a bounded default and a hard cap). Validation errors raise `AppError` subclasses; do not raise `HTTPException`.
- `app/api/__init__.py` re-exports the two new public symbols: `SecurityHeadersMiddleware` and `CursorParams` (plus the pagination dependency callable). No other symbols; no side-effect imports.

Tests required

- Middleware unit test on a **bare Starlette app** (no `create_app`): every listed header is present on the response with the exact required values; when `security_headers_enabled=False`, no headers are added.
- Pagination dependency: default `limit`, hard cap enforcement, invalid `cursor` handling via `AppError` (not `HTTPException`).
- CORS behavior is **not** tested here; CORS tests live in T-505.

Acceptance criteria

- `SecurityHeadersMiddleware` and `CursorParams` (and the pagination dependency) are importable from `app.api`.
- The middleware is a pure ASGI class with header-emission unit tests; it is **not** registered on any FastAPI app in this task.
- CORS is not configured in this task; it is registered by `app_factory` (T-505) using the `Api` settings group from T-301.

Commands: standard four.

Common failure modes: reading settings directly inside the middleware; registering the middleware here; adding CORS logic; raising `HTTPException` from `pagination.py`.

Review checklist: two public symbols re-exported from `app.api`, all five headers present, middleware not registered anywhere, no CORS logic, pagination raises `AppError` only.
