Purpose: `GET /api/v1/users/me`.

Depends on: T-906, T-906C.

Allowed files: `app/users/{__init__,domain,persistence,service,api,deps}.py` + tests + migration `0003_users.py` (profile columns separate from `users` auth fragment; or share table if your migration in T-905 already covers it ÔÇö choose **share**, owning the table from `users` since auth only references `user_id`) + `app/api/v1.py` (append-only). In `app/api/v1.py`, **append** exactly the single line `router.include_router(users.api.router, prefix="/users", tags=["users"])` to `build_v1_router()` (per rules.md ┬ž1 "Append-only edits to shared assembly files" and "The `/api/v1` router is incremental"). Do not modify any other line of `app/api/v1.py`. The effective route prefix is `/api/v1/users`.

Forbidden: `app.users` must not import `app.auth.persistence` or `app.auth.adapters` (rule ┬ž3). Read-only domain types from `app.auth` only.

Implementation requirements: `User` profile dataclass; `GET /api/v1/users/me` returns the current user's profile.

Profile creation in Phase 2 uses the **lazy-read** path (per forward-audit patch P-8): `users.service.get_or_create_profile(user_id, email=current_user.email)` is invoked from `users.api` on first `GET /users/me` for that authenticated principal. If no row exists yet in the `users` profile table, the service inserts a minimal profile record inside the same transaction and returns it. Subsequent calls read the existing row. This preserves the dep-graph constraint (`auth Ôćĺ users` remains forbidden) without introducing any cross-module event bus.

Explicitly out of scope for Phase 2 (deferred to Phase 3): `app.shared.events` in-process pub/sub, `UserRegistered` events, an `auth.service.register`-driven push into `users`, and any `UserProfileWriter` port. Do not create `app/shared/events.py` from this task. If a future feature needs eager profile creation, it will be introduced by a dedicated Phase 3 ADR and its own task.

Tests required: 401 unauthenticated; 200 with profile when authenticated; first `GET /users/me` for a freshly registered principal creates the profile row lazily and a subsequent call returns the same row without duplicating it.

Acceptance: dep-graph contract still passes (`auth Ôćĺ users` forbidden).

Otherwise standard.
