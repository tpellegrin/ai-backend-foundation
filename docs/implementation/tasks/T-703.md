# Task T-703 — `app/infrastructure/http/` shared httpx

> Section: **S07 — Infrastructure base**
>
> Before you start, read [`../rules.md`](../rules.md) (global rules, stop signals, allowed-files discipline, command discipline, lower-complexity model instructions).
> After you finish, self-check against [`../review.md`](../review.md) (final review checklist, architecture compliance).

---


Purpose: one configured `AsyncClient` for every external HTTP call (rule §7 of AGENTS.md).

Depends on: T-301.

Allowed files: `app/infrastructure/http/{__init__,client}.py`, `app/infrastructure/http/tests/test_client.py`.

Forbidden: do not import `requests` or `urllib.request`. **Do not import `opentelemetry.*` from this package** — that would violate rules.md §1 C-2 ("OTel exporters and providers are constructed inside `app/core/wiring/`. `app/observability/` only exposes config types, middleware factories, and the `request_id_var` context var."). `app/infrastructure/http/` must remain OTel-free at the import level; instrumentation is applied globally by the wiring layer (see below).

Implementation requirements

- Factory `build_http_client(settings: AppSettings) -> AsyncClient` builds an `httpx.AsyncClient` with an `httpx.Timeout(connect=settings.http.connect_timeout_s, read=settings.http.read_timeout_s, write=settings.http.read_timeout_s, pool=settings.http.total_timeout_s)`. No OTel imports here.
- Retry policy: `tenacity` (already an allowed dep in T-703's scope) with `stop_after_attempt(settings.http.retry_max_attempts)`, `wait_exponential_jitter(initial=settings.http.retry_backoff_factor)`, retrying only on `httpx.ConnectError`, `httpx.ReadTimeout`, and `httpx.HTTPStatusError` where `status_code >= 500`. Never retry 4xx; never retry non-idempotent verbs unless the caller opts in.
- Typed helper `request_json(client, method, url, ...) -> dict[str, Any]` returning parsed JSON or raising `UpstreamProviderError` (defined in this package's `client.py`) for retry-exhausted or 4xx responses.
- **OTel instrumentation is out of scope for this task.** `HTTPXClientInstrumentor().instrument()` is a global (module-level) side effect that patches `httpx` process-wide. It is invoked exactly once at startup by an `apply_httpx_instrumentation(settings)` helper that lives at `app/core/wiring/observability.py`. Per T-403's note ("Exporter providers are installed by `app/core/wiring/observability.py` (added implicitly during T-504/T-505 wiring)") and rules.md §1 C-2, that file — and the instrumentation helper — is materialized by the S05 wiring tasks (T-504/T-505), not by T-703 and not by T-403. T-703 exposes the plain factory; the wiring helper invokes `HTTPXClientInstrumentor().instrument()` once during app startup so every client subsequently produced by `build_http_client` is auto-instrumented without T-703 (or `app/observability/`) importing any OTel symbol. If `app/core/wiring/observability.py` does not yet exist when T-703 lands, this task **stops and reports** — do not add the OTel side effect here, and do not create the wiring helper from this task.

Tests required: respx-backed test asserts retries on 503 (up to `settings.http.retry_max_attempts`), surfaces `UpstreamProviderError` for 4xx that should not retry, and honors `settings.http.connect_timeout_s`. **Do not** assert on OTel spans in this task's tests — instrumentation is covered by T-403's tests.

Acceptance: every provider adapter (S07/S12) uses this client; no direct `httpx.AsyncClient()` elsewhere; grep for `opentelemetry` under `app/infrastructure/http/` returns zero matches.

Commands: standard four.

Common failure modes: retrying non-idempotent verbs; swallowing 4xx; importing `opentelemetry` inside this package to "enable tracing" (forbidden — see rules.md §1 C-2).

Review checklist: timeouts from `settings.http.*`, retries bounded by `settings.http.retry_max_attempts`, error wrapper present, **no `opentelemetry` import anywhere under `app/infrastructure/http/`**.
