# Task T-703 — `app/infrastructure/http/` shared httpx

> Section: **S07 — Infrastructure base**
>

> After you finish, self-check against [`../review.md`](../review.md) (final review checklist, architecture compliance).

---


Purpose: one configured `AsyncClient` for every external HTTP call (rule §7 of AGENTS.md).

Depends on: T-301.

Allowed files: `app/infrastructure/http/{__init__,client}.py`, `app/infrastructure/http/tests/test_client.py`, `app/core/wiring/http.py`, `app/core/wiring/observability.py`, `app/core/wiring/tests/test_observability_httpx.py`, `app/main/app_factory.py`.

Rationale for `app/main/app_factory.py`: the HTTPX instrumentation helper must be invoked exactly once at the composition root (ADR-0030).

Forbidden: do not import `requests` or `urllib.request`. **Do not import `opentelemetry.*` from this package** — that would violate rules.md §1 C-2 ("OTel exporters and providers are constructed inside `app/core/wiring/`. `app/observability/` only exposes config types, middleware factories, and the `request_id_var` context var."). `app/infrastructure/http/` must remain OTel-free at the import level; instrumentation is applied globally by the wiring layer (see below).

Implementation requirements

- **Settings boundary**: `app.infrastructure.http.client` must not import `AppSettings` or any module from `app.core.*`. It defines a local `HttpClientSettings` (as a `Protocol` or simple `dataclass`) containing only the primitives it needs: `connect_timeout_s`, `read_timeout_s`, `total_timeout_s`, `retry_max_attempts`, and `retry_backoff_factor`.
- Factory `build_http_client(settings: HttpClientSettings) -> AsyncClient` builds an `httpx.AsyncClient` with an `httpx.Timeout(connect=settings.connect_timeout_s, read=settings.read_timeout_s, write=settings.read_timeout_s, pool=settings.total_timeout_s)`. No OTel imports here.
- Retry policy: `tenacity` (already an allowed dep in T-703's scope) with `stop_after_attempt(settings.retry_max_attempts)`, `wait_exponential_jitter(initial=settings.retry_backoff_factor)`, retrying only on `httpx.ConnectError`, `httpx.ReadTimeout`, and `httpx.HTTPStatusError` where `status_code >= 500`. Never retry 4xx.
- **Idempotency**: Retrying non-idempotent verbs (POST, PATCH) requires an explicit `retry_non_idempotent: bool = False` opt-in parameter in the helper.
- Typed helper `request_json(client, method, url, ...) -> dict[str, Any]` returning parsed JSON or raising `UpstreamProviderError`. It must use explicit parameters for common needs (headers, json, params) instead of an unbounded `**kwargs: Any` boundary.
- `UpstreamProviderError` is owned by `app.infrastructure.http.client` for this task.
- **HTTPX instrumentation wiring is owned by this task.** T-703 introduces `app.core.wiring.observability.apply_httpx_instrumentation(settings: AppSettings)`. This helper owns the HTTPX instrumentation side effect for the HTTP client factory. It reads the full `AppSettings` to determine if instrumentation should be enabled. This is the only file allowed to import `opentelemetry.instrumentation.httpx`. It is invoked exactly once from `app/main/app_factory.py`.
- `app.core.wiring.http` is introduced to manage the construction of the client and the adaptation of `AppSettings.http` to `HttpClientSettings`.
- `app/observability/*` remains provider/instrumentation-import free unless already explicitly allowed elsewhere.
- T-703 does not own general tracer/meter provider installation or other OTel instrumentations. It owns HTTPX instrumentation wiring only.

Tests required: respx-backed test asserts retries on 503 (up to `settings.http.retry_max_attempts`), surfaces `UpstreamProviderError` for 4xx that should not retry, and honors `settings.http.connect_timeout_s`. **A dedicated test in `app/core/wiring/tests/test_observability_httpx.py` verifies that `apply_httpx_instrumentation` correctly patches httpx.**

Acceptance: `apply_httpx_instrumentation` is called in `app_factory.py`; grep for `opentelemetry` under `app/infrastructure/http/` returns zero matches. New provider adapters must use this client; direct `httpx.AsyncClient()` construction outside approved client/wiring/tests is forbidden.

Commands: standard four.

Common failure modes: retrying non-idempotent verbs; swallowing 4xx; importing `opentelemetry` inside this package to "enable tracing" (forbidden — see rules.md §1 C-2).

Review checklist: timeouts from `settings.http.*`, retries bounded by `settings.http.retry_max_attempts`, error wrapper present, **no `opentelemetry` import anywhere under `app/infrastructure/http/`**.
