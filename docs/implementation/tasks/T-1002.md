## Implementation requirements

### Prompt YAML

Create:

```text
app/prompts/library/rag_answer_v1.yaml
```

The YAML must declare at least:

```yaml
id: rag_answer
version: 1.0.0
description: Canonical RAG answer prompt for Phase 2.
owner: app.prompts
confidential: false
input_schema: app.prompts.library.rag_answer_v1:RagAnswerInput
output_schema: app.prompts.library.rag_answer_v1:RagAnswerOutput
template: |
  ...
```

The template must be Jinja2-compatible and should include:

- a system-style instruction section
- a user question section using `{{ question }}`
- a citations/context section that loops over `citations`
- citation fields:
  - `document_id`
  - `chunk_id`
  - `source`
  - `text`
- clear instructions to answer only from supplied context
- clear instructions to avoid inventing unsupported facts
- clear instructions to return a concise answer suitable for later structured output validation

Keep the prompt practical and minimal. Do not turn this into a full RAG product policy, citation formatter, or safety framework.

### Pydantic schemas

Create:

```text
app/prompts/library/rag_answer_v1.py
```

Define Pydantic models:

```python
from app.shared.pydantic import BaseSchema

class CitationInput(BaseSchema):
    document_id: str
    chunk_id: str
    source: str
    text: str

class RagAnswerInput(BaseSchema):
    question: str
    citations: list[CitationInput]

class RagAnswerOutput(BaseSchema):
    answer: str
```

Use `app.shared.pydantic.BaseSchema` for all prompt-related schemas.

Validation should be minimal but useful:

- require non-empty `question`
- require at least one citation
- require non-empty citation text
- require non-empty answer

Use Pydantic v2-compatible validation patterns already used in the project.

## Acceptance criteria

- `rag_answer_v1.yaml` declares `id: rag_answer`.
- `rag_answer_v1.yaml` declares `version: 1.0.0`.
- `rag_answer_v1.yaml` declares `input_schema: app.prompts.library.rag_answer_v1:RagAnswerInput`.
- `rag_answer_v1.yaml` declares `output_schema: app.prompts.library.rag_answer_v1:RagAnswerOutput`.
- The prompt template includes placeholders for `question` and a loop over `citations`.
- `RagAnswerInput`, `CitationInput`, and `RagAnswerOutput` inherit from `BaseSchema`.
- Tests render the prompt with representative input and assert the rendered text contains the question and citation text.
- Tests validate a representative sample answer against `RagAnswerOutput`.
- The prompt lives at `app/prompts/library/rag_answer_v1.yaml`.
- The prompt schemas live at `app/prompts/library/rag_answer_v1.py`.
- The YAML references schemas using `module.path:ClassName` strings.
- No prompt text is duplicated in Python source.
- The shipped prompt can be loaded by the T-1001 registry.
- The shipped prompt renders with valid sample input.
- The shipped output schema validates a representative answer.
- No RAG orchestration, LLM call, provider integration, vector-store integration, or API route is added.
- `make lint`, `make typecheck`, `make test`, and `uv run lint-imports` pass.
