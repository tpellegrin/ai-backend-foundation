## Purpose

Implement the Auth API router for the auth endpoints:

- `register`
- `login`
- `refresh`
- `logout`

Mount the Auth API router into the incremental `/api/v1` router.

This task changes the expected behavior of `build_v1_router()`: after T-907, the v1 router is no longer empty because it includes the auth router.

## Implementation requirements

Use Pydantic v2 request/response models in `app/auth/api.py`.

Raise `AppError` subclasses only.

Declare Problem Details response shapes through `responses=`.

The module router itself must be declared with no prefix. The `/auth` prefix is applied at the mount site.

In `app/api/v1.py`, append exactly the single line below to `build_v1_router()`:

```python
router.include_router(auth.api.router, prefix="/auth", tags=["auth"])
```

This follows `rules.md` §1, especially:

- append-only edits to shared assembly files;
- the `/api/v1` router is incremental.

Do not modify any other line of `app/api/v1.py`.

The effective route prefix is:

```text
/api/v1/auth
```

## Test update required

Update `app/api/tests/test_v1_router.py` to reflect the new intended router behavior.

The previous T-503 expectation that `build_v1_router()` returns an empty router is obsolete after this task, because T-907 mounts the auth router.

The updated test should verify:

- `build_v1_router()` still returns a router with the expected `/v1` prefix;
- the router is no longer empty;
- the auth router/route is included.

Prefer an explicit route assertion over a brittle route-count-only assertion.

Example intent:

```python
def test_build_v1_router_includes_auth_router_with_prefix() -> None:
    router = build_v1_router()

    assert router.prefix == "/v1"
    assert router.routes
```

If the project’s route inspection style makes it practical, assert the auth route path or route name explicitly rather than only checking `router.routes`.

Do not use this permission to rewrite unrelated v1 router tests.
