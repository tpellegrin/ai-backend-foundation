Implementation requirements: implements the T-601 port surface — every method accepts `bucket: str` as its first argument. The adapter constructor must not take `AppSettings`; it must take the `base_path: Path` to the storage root (adapted from `settings.blob_storage.local_dir` by the wiring layer). `put(bucket, key, data, content_type)` writes to `base_path/<bucket>/<key>`, creates parent directories, and returns a `BlobRef` populated with the effective bucket/key/content_type/size. `get(bucket, key)` streams bytes back. `delete(bucket, key)` removes the file (idempotent — missing file is not an error). `presign_get(bucket, key, ttl_s)` returns a `file://` URI (dev only); `ttl_s` is accepted for signature compatibility but ignored.

Key/bucket safety (validated **before** any filesystem touch, raising `ValueError`):

- Reject any segment equal to `..` (parent-directory traversal), including when combined with forward or backward slashes (e.g. `..`, `a/../b`, `..\\b`).
- Reject leading `/` absolute POSIX paths (e.g. `/etc/passwd`).
- Reject Windows drive prefixes and UNC-style prefixes (e.g. `C:\\foo`, `c:/foo`, `\\\\server\\share`).
- Reject any backslash (`\\`) anywhere in `bucket` or `key` — treat backslashes as path separators for validation purposes even on POSIX.
- After joining `base_path / bucket / key` and calling `Path.resolve()`, reject any result whose resolved path is not a descendant of the resolved storage root (final containment check).
- Reject empty strings, `.` segments, and NUL bytes in either `bucket` or `key`.

Tests required: unit + contract suite for `BlobStorage` runs against the local adapter. Unit tests must cover each rejection case above (traversal, leading `/`, Windows drive prefix, backslash, symlink-or-otherwise-escaping normalized path, empty/`.`/NUL).

Acceptance: passes the BlobStorage contract suite (added next to ports tests); all key/bucket safety cases above are rejected with `ValueError` and never touch the filesystem.

Commands: standard four.

Common failure modes: path traversal via `../`; accepting backslash on POSIX; missing final containment check after `Path.resolve()`; missing dir creation.

Review checklist: sanitized keys per the full safety list above; final resolved-path containment check present; contract test green.
