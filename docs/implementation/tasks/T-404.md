# Task T-404 — `middleware.py` access log

> Section: **S04 — Observability**

Purpose: emit a single structured access log per request. **Security headers are out of scope**; they are owned by T-502 (`app/api/security_headers.py`). Do not add header logic here.

Depends on: T-401, T-402.

Allowed files: `app/observability/middleware.py`, `app/observability/tests/test_middleware.py`.

Implementation requirements: `AccessLogMiddleware` logs method, path, status, duration_ms, request_id, user_id (if available). Reads no app state directly. Emits exactly one log event per request via `structlog`. Does **not** set, read, or modify HTTP response headers — that responsibility belongs to T-502.

Tests required: middleware emits one event per request with required keys (`method`, `path`, `status`, `duration_ms`, `request_id`); no security headers are set by this middleware (delegate assertion belongs to T-502's tests).

Acceptance criteria: zero ad-hoc logging in route handlers for access-style events; no security-header side effects introduced by this task.

Commands: standard four.

Common failure modes: logging twice; reading `request.body` (consumes the stream); reaching into T-502's security-headers scope.

Review checklist: single emission, no body reads, no header manipulation.
