Purpose: produce a configured FastAPI app for serving and for tests.

Depends on: T-501..T-504.

Allowed files: `app/core/app_factory.py`, `app/core/tests/test_app_factory.py`.

Implementation requirements

- `create_app(settings: AppSettings | None = None) -> FastAPI`:
  - configures logging (via T-401),
  - installs middleware in this exact order (outermost first): `CorrelationMiddleware` (T-402) → `AccessLogMiddleware` (T-404) → `SecurityHeadersMiddleware` (T-502; construct with `settings.api.security_headers_enabled`) → `starlette.middleware.cors.CORSMiddleware` configured **from the `Api` settings group** (T-301: `cors_allowed_origins`, `cors_allow_credentials`, `cors_allowed_methods`, `cors_allowed_headers`),
  - mounts the `/api/v1` router from T-503 (`build_v1_router()`) and the health router from T-405 (`build_health_router(registry=container.probe_registry, is_ready=lambda: app.state.ready)`),
  - registers exception handlers via `register_exception_handlers` (T-501),
  - sets the lifespan from `core.lifespan` (T-504), which owns `app.state.ready` and populates the probe registry on the `Container`.
- CORS is registered here — not in T-502. When `cors_allowed_origins` is empty, `CORSMiddleware` is still installed but denies every cross-origin request (deny-by-default).

Tests required

- App boots in-process with a stubbed container; `/api/v1` and health routes are registered.
- **Middleware registered once**: no middleware class appears more than once in `app.user_middleware`.
- **CORS deny-by-default**: with `cors_allowed_origins=()`, a cross-origin preflight from any origin does not receive `Access-Control-Allow-Origin`.
- **CORS whitelist**: with `cors_allowed_origins=("https://ok.example",)`, a preflight from `https://ok.example` succeeds and any other origin is denied.
- **Security headers**: every response carries all five headers listed in T-502.
- **Readiness semantics**: `/readyz` returns 503 before lifespan startup completes and 200 after; `/livez` returns 200 in both states.

Acceptance criteria: `app.main:app = create_app()`; middleware, CORS, and both routers are registered exactly once from this task.

Commands: standard four.

Common failure modes: building two `FastAPI` instances; middleware order wrong (correlation must wrap access log).

Review checklist: middleware order, lifespan attached, no duplicate handlers.
