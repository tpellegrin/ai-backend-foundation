# Task T-701 — `app/infrastructure/db/`

> Section: **S07 — Infrastructure base**
>
> Before you start, read [`../rules.md`](../rules.md) (global rules, stop signals, allowed-files discipline, command discipline, lower-complexity model instructions).
> After you finish, self-check against [`../review.md`](../review.md) (final review checklist, architecture compliance).

---


Purpose: async SQLAlchemy engine, sessionmaker, declarative `Base`, pgvector type registration, and the DB wiring that appends `db_engine`/`session_factory` to the shared `Container` plus a DB readiness probe.

Depends on: T-301, T-203, T-504.

Allowed files: `app/infrastructure/__init__.py`, `app/infrastructure/db/__init__.py`, `app/infrastructure/db/engine.py`, `app/infrastructure/db/base.py`, `app/infrastructure/db/types.py`, `app/infrastructure/db/tests/test_engine.py`, `app/core/wiring/__init__.py`, `app/core/wiring/db.py`, `app/core/wiring/tests/test_wiring_db.py`, `app/core/container.py` (append-only), `app/core/di.py` (append-only), `app/core/lifespan.py` (append-only), `app/main/tests/test_app_factory.py`, `tests/integration/test_lifespan.py`, `.importlinter` (single-line addition, per [ADR-0025](../../adr/0025-direct-import-semantics-for-core-wiring-only-infra.md)).

> T-701 is the **first** task that materializes the `app/core/wiring/` package and therefore the **first** task that introduces the `app.core.wiring.* → app.infrastructure.*` edge (via `app/core/wiring/db.py` importing `app.infrastructure.db.*`). It must create `app/core/wiring/__init__.py` (empty is fine) so the package exists before `db.py` is imported.
>
> **`.importlinter` change owned by this task ([ADR-0025](../../adr/0025-direct-import-semantics-for-core-wiring-only-infra.md)).** T-701 owns the one-line flip on the `core-wiring-only-infra` `forbidden` contract: add `allow_indirect_imports = True`. This clarifies the contract's semantics to a **direct-import ban only**, so that the sanctioned ADR-0023 composition flow `app.main.app_factory → app.core.lifespan → app.core.wiring.db → app.infrastructure.db` passes `lint-imports` without weakening or ignoring any edge. Do not modify `source_modules` or `forbidden_modules`. Do not add any `ignore_imports` entries. Do not modify any other contract. Do not weaken any other import-linter contract in this task.
>
> **Forbidden workarounds ([ADR-0025](../../adr/0025-direct-import-semantics-for-core-wiring-only-infra.md)).** The wiring edge from `app.core.lifespan` to `app.core.wiring.db` must be a **static, top-of-file** `from app.core.wiring.db import <symbol>` import. `importlib.import_module("app.core.wiring.db")` in `lifespan.py`, function-local imports of `app.core.wiring.*`, and any re-export shim intended to hide the composition edge from Import Linter or `mypy --strict` are all forbidden. A previous T-701 attempt used a dynamic import here; review rejected it and the reviewer must reject it again if reintroduced.
>
> **Scope expansion rationale**: T-701 is the first task to register an external-I/O readiness probe (`DBProbe`). Pre-existing tests in `app/main/tests/test_app_factory.py` and `tests/integration/test_lifespan.py` that asserted `/readyz == 200` against an empty `ProbeRegistry` must be updated in lockstep to avoid regressions in environments without a live database.

Forbidden: do not import `psycopg2`; do not create sessions outside this package (rule 4 §7 of AGENTS.md). No business logic. `app.infrastructure.db.*` may be imported **only** from `app/core/wiring/db.py`. Edits to `container.py`, `di.py`, and `lifespan.py` are strictly append-only per rules.md §1 "Append-only edits to shared assembly files" — no reordering or rewriting of fields/hooks/providers landed by T-504.

Implementation requirements:

- `create_engine_from(settings) -> AsyncEngine`; `async_sessionmaker[AsyncSession]`; `Base = DeclarativeBase`; `types.py` registers `pgvector.sqlalchemy.Vector`. Append a session-per-request FastAPI dependency helper to `app/core/di.py` (e.g. `get_db_session(container) -> AsyncIterator[AsyncSession]`) that opens an `AsyncSession` from the container's `session_factory` and closes it on request teardown. This edit to `di.py` is strictly append-only.
- `app/core/wiring/db.py` constructs the async engine and session factory from settings and exposes a DB readiness probe (a cheap `SELECT 1` against the engine).
- Append to `app/core/container.py`: `db_engine: AsyncEngine | None = None` and `session_factory: async_sessionmaker[AsyncSession] | None = None` (Optional / defaulted per rules.md §1 incremental-Container rule).
- Append to `app/core/lifespan.py`: (a) a startup hook that builds the engine + session factory and populates the two container fields, (b) a shutdown hook that awaits `engine.dispose()`, (c) the DB readiness probe registered on the `Container`'s `ProbeRegistry` (per rules.md §1 incremental-probe rule).

Tests required (integration): connect to Testcontainers Postgres+pgvector and `SELECT 1`; wiring smoke test confirms the two new container fields are populated after lifespan startup and that the DB readiness probe is present in the registry.

Acceptance criteria: domain modules never construct `AsyncSession` themselves; only `core.wiring.db` imports `app.infrastructure.db.*`; container edits are strictly append-only.

Commands: standard four + `make test-int`.

Common failure modes: synchronous engine; missing pgvector extension before type usage; opening a second engine outside the shared lifespan; rewriting rather than appending to `container.py`/`lifespan.py`.

Review checklist: async engine, Base provided, pgvector registered; `db_engine`/`session_factory` appended to `Container`; DB probe appended to registry; lifespan hooks are additive.
