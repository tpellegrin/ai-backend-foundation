# Task T-506 — `app/main/__init__.py` ASGI entrypoint

> Section: **S05 — App factory + API edge**
>
> Before you start, read [`../rules.md`](../rules.md) (global rules, stop signals, allowed-files discipline, command discipline, lower-complexity model instructions).
> After you finish, self-check against [`../review.md`](../review.md) (final review checklist, architecture compliance).

---


Purpose: ASGI entrypoint.

Depends on: T-505.

Allowed files: `app/main/__init__.py`, `tests/test_main.py`.

Assumed pre-existing repository fixture: `tests/conftest.py` (repository-wide test env bootstrap, per [`../rules.md`](../rules.md) §1 *Repository-wide test env bootstrap* and §4). T-506 **must not** create, modify, or delete this file. It is guaranteed to seed the minimal placeholder environment `AppSettings` needs at pytest collection time; the eager `app = create_app()` binding below relies on that guarantee.

Implementation requirements: `from app.main.app_factory import create_app` then `app = create_app()`. Nothing else. Per [ADR-0023](../../adr/0023-composition-root-ownership.md), the composition root lives in `app.main`, not `app.core`; the factory module is `app.main.app_factory` from T-505. The eager `app = create_app()` binding at import time is **intentional** — `uvicorn app.main:app` and every ASGI-attach path depend on it, and pytest collection is protected by the repository-wide `tests/conftest.py` bootstrap. Do not defer construction behind a getter, a lazy proxy, an `if __name__ == "__main__":` guard, an `os.getenv` sentinel, or `importlib.reload` gymnastics.

Tests required (`tests/test_main.py`): exactly one `@pytest.mark.unit` test asserting that `import app.main` succeeds and that `app.main.app` is an instance of `fastapi.FastAPI`. Do not add integration or subprocess-based tests here — end-to-end serving is covered by T-505's factory tests and T-507's compose smoke, not by this two-line entrypoint. The test **must not**:
- call `os.environ[...] = ...`, `os.environ.copy()`, `os.setenv`, or otherwise mutate `os.environ` (`../rules.md` §3 rule 8);
- spawn a subprocess Python interpreter (`subprocess.run(sys.executable, ...)`) — no supported test kind under `AGENTS.md` §5 / `rules.md` §4;
- call `importlib.reload(app.main)` — reload on the entrypoint mutates module identity and side-effects other tests in the same session;
- rely on any environment beyond what the repository-wide `tests/conftest.py` guarantees.

Acceptance criteria: `uvicorn app.main:app` serves; `import app.main` succeeds under `make test` given the standard repository-wide test bootstrap.

Commands: standard four.

Common failure modes:
- Putting routes, middleware, or lifespan logic in `app/main/__init__.py` — all of that belongs to T-505's `create_app()`.
- Reading env directly (`os.getenv`, `os.environ`) — `rules.md` §3 rule 8.
- Trying to make `app = create_app()` "collection-safe" by adding a lazy proxy, module-level try/except, autouse env fixture in `tests/test_main.py`, or duplicated env-dict literals. The repository-wide `tests/conftest.py` already guarantees collection safety; a per-task workaround is a `rules.md` §3 rule 8 / §4 violation and a review-BLOCKING defect (see [`../review.md`](../review.md) §4).
- Adding `tests/conftest.py` to `Allowed files` — the file is pre-existing structural scope, not per-task scope. Only a future task that introduces a new **required** `AppSettings` field may list it (append-only, one `setdefault` line).

Review checklist: 2–3 lines only. (a) `app/main/__init__.py` is exactly the two mandated lines; (b) `tests/test_main.py` contains a single unit test asserting `isinstance(app.main.app, FastAPI)` with no env mutation, no subprocess, no reload; (c) `tests/conftest.py` is unmodified.
