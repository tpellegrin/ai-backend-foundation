# Task T-906C — Propagate email through authenticated principal

> Section: **S09 — Auth & Users / auth-users contract bugfix**

## Purpose

Expose the authenticated user's email through the public auth principal returned by `get_current_user`, so `T-910` can implement `GET /api/v1/users/me` without importing auth persistence, auth adapters, or decoding JWTs in `app.users`.

## Forbidden changes

- Do not modify `app/users/*`.
- Do not implement `GET /api/v1/users/me`.
- Do not make `app.users` import `app.auth.persistence`.
- Do not make `app.users` import `app.auth.adapters`.
- Do not decode JWTs in `app.users`.
- Do not use placeholder emails.
- Do not add SQLAlchemy logic outside `app/auth/persistence.py`.
- Do not leak ORM rows outside persistence.
- Do not modify wiring unless current-user dependency retrieval is impossible without it.

## Implementation requirements

- Add `email` to `AuthenticatedUser` or introduce an equivalent public authenticated principal type in `app.auth.domain`.
- Preserve the existing identifier field name on `AuthenticatedUser`; add `email` without renaming existing public fields.
- Update auth service/dependency logic so `get_current_user` returns the authenticated user's real email.
- For this task, propagate email through verified access-token claims.
- Access tokens issued by register/login/refresh flows must include the authenticated user's email as a claim. `get_current_user` must verify the token and construct `AuthenticatedUser` from trusted claims.
- Do not add a DB lookup to `get_current_user` in this task unless existing production code already performs one. If a DB-backed current-user lookup is desired later, create a separate task.
- Preserve existing token validation and disabled-user behavior.
- `AuthenticatedUser` should contain only public authenticated-principal fields needed by downstream modules, such as `id`/`user_id`, `email`, and possibly `tenant_id` if already present. It must not contain `password_hash`, refresh-token fields, disabled-state internals, or ORM row objects.
- Update T-910 so it explicitly depends on this task and uses `current_user.email`.

- Making users decode JWTs.
- Making users import auth persistence.
- Adding password_hash to `AuthenticatedUser`.
- Fixing only test overrides while production `get_current_user` still lacks email.
