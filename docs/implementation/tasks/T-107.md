Purpose: encode the Phase 2 dependency graph (¬ß03 of revision pack) into machine-checked contracts before any module exists, so future tasks cannot drift.

Depends on: T-102.

Allowed files: `.importlinter`, `app/__init__.py`, `tests/__init__.py`, `tests/test_imports.py`.

Forbidden: do not weaken or remove a contract to make a future task pass. Do not add `ignore_imports` patterns beyond those explicitly enumerated in ¬ß03. Do not create any other `app/*/__init__.py`, `app/*/<file>.py`, or `tests/<other>.py` files ‚Äî even an empty `app/shared/__init__.py` is out of scope for T-107 and belongs to its owning task. Explicitly forbid creating any future package marker just to satisfy Import Linter. Do not move the config into `pyproject.toml` (T-102 owns `pyproject.toml` and must not be edited here).

Config-file decision (binding)

- Use **`.importlinter`** at the repository root, **INI format**. Rationale:
  - import-linter auto-discovers exactly three filenames: `setup.cfg` (INI), `.importlinter` (INI), `pyproject.toml` (TOML, under `[tool.importlinter]`). A standalone `importlinter.toml` is **not** auto-discovered, and even with `--config importlinter.toml` it is only read when its sections live under `[tool.importlinter]` ‚Äî which would conflict with the file's name and add a `--config` flag to every invocation site (Makefile, pre-commit, CI, tests). See `import-linter` `adapters/user_options.py` (`IniFileUserOptionReader.potential_config_filenames`, `TomlFileUserOptionReader.section_name = "importlinter"` keyed under `tool.`).
  - `.importlinter` keeps the config out of `pyproject.toml` (owned by T-102) and lets every invocation site use the bare command `uv run lint-imports` (no `--config` flag, no drift between Makefile / pre-commit / CI / tests).

Note on early-repository state: `lint-imports` needs at least one importable root package to build the graph. This task creates an **empty** `app/__init__.py` (and `tests/__init__.py`) so contracts pass cleanly on the skeleton. With an empty `app/` package, every `forbidden` / `independence` / `layers` contract is vacuously satisfied because no imports yet exist ‚Äî this is the intended early-state behavior, formalized in `rules.md` ¬ß2. The same contracts continue to apply unchanged after every later task adds modules. Do **not** scaffold `app.shared`, `app.observability`, `app.platform`, etc. early to "make layers non-empty" ‚Äî instead, use optional layer syntax `(app.<name>)` in `.importlinter` for not-yet-existing packages.

Implementation requirements

- File is `.importlinter` (INI format) at repo root.
- `[importlinter]` section with `root_packages = app` (single root; one per line if multiple ‚Äî none in Phase 2).
- Add a comment in `.importlinter` explaining:
    - missing future packages are represented as optional layers `(app.<name>)`;
    - future `ignore_imports` entries must be added only when the matching import exists;
    - dummy imports and dummy package markers are forbidden.
- Each contract in its own `[importlinter:contract:<id>]` section. Required contracts:
  - "Layers" ‚Äî `type = layers`, exact layer order from `docs/phase-2-revision/03-revised-dependency-graph.md ¬ß1` (top to bottom: `app.main`, `app.api`, `app.core`, domains as a `|`-grouped layer, capabilities as a `|`-grouped layer, `app.platform`, `app.observability`, `app.shared`). Use optional layer syntax `(app.<name>)` for packages that do not yet exist.
  - "Capabilities are independent" ‚Äî `type = independence` over `app.llm`, `app.embeddings`, `app.prompts`.
  - "ai and rag are independent" ‚Äî `type = independence` over `app.ai`, `app.rag`.
  - "auth does not import users" ‚Äî `type = forbidden`.
  - "Only core.wiring imports infrastructure" ‚Äî `type = forbidden`; sources include every module except `app.core.wiring.*`; `forbidden_modules = app.infrastructure.**`. Enforces that only the composition root may import adapters.
  - "Infrastructure adapter-ring boundary" ‚Äî `type = forbidden`; enforces that `app.infrastructure.**` does not import `app.main`, `app.api`, or `app.core`.
  - "Infrastructure adapters are independent" ‚Äî `type = independence`; enforces that adapters do not import each other.
  - "Platform is below everything except shared" ‚Äî `type = forbidden`; sources = `app.platform`; forbidden = list from ¬ß03.
  - "No cross-module persistence/adapters imports" ‚Äî `type = forbidden`; with the exact `ignore_imports` whitelist from ¬ß03.

Tests required

- `tests/test_imports.py::test_import_linter_passes` shells out to `uv run lint-imports` (no `--config` flag ‚Äî rely on auto-discovery of `.importlinter`) and asserts `returncode == 0`. This test simultaneously validates that the config file is loaded (it would otherwise fail with `FileNotFoundError: Could not read any configuration.`) and that all contracts pass on the empty skeleton.

Acceptance criteria

- `uv run lint-imports` exits 0 on the empty skeleton (`app/__init__.py` only) **and** prints a contract summary (not "Could not read any configuration."). The same command remains exit 0 after every later task.
- No file or directory under `app/` exists other than `app/__init__.py` after this task runs.

Commands

```
uv run lint-imports
uv run pytest tests/test_imports.py -q
```

Common failure modes:

- Naming the config `importlinter.toml` ‚Äî not auto-discovered, requires `--config`, and the standalone-TOML reader expects `[tool.importlinter]` (not `[importlinter]`). Both invariants must hold or load fails silently with `FileNotFoundError: Could not read any configuration.` even when `--config` is passed. Use `.importlinter` (INI) instead.
- Adding `--config <path>` to `uv run lint-imports` in Makefile / pre-commit / CI / tests ‚Äî the bare command must work via auto-discovery; an explicit flag would mask a missing or renamed file.
- Silencing a contract with broad `ignore_imports` instead of fixing the offending edge.
- Renaming `app.infrastructure` to dodge the contract.
- Pre-loading future `ignore_imports` entries. Add ignore exceptions only in the same task that introduces the corresponding import edge.

Review checklist: 7 contracts present, names match ¬ß03, no pre-loaded `ignore_imports` (only added when matching imports exist), file is `.importlinter` (INI) at repo root, no future package markers created (use optional layers instead).
