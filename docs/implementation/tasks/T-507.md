Purpose: enforce Problem Details and X-Request-ID echo at the edge.

Depends on: T-501, T-505.

Allowed files: `tests/api/test_problem_details.py`, `tests/api/test_correlation.py`.

Implementation requirements
- Boot the app with a stub container (no DB) and assert:
  - Triggering a known `AppError` returns a 4xx Problem Details with all required fields.
  - An unhandled exception returns a sanitized 500.
  - Each response echoes inbound `X-Request-ID`; absent → generated.
  - Content-Type is `application/problem+json` on errors.

- **Trigger mechanism (per forward-audit patch P-12).** At S05 no module `api.py` yet exists, so the test itself must install the routes it exercises. Inside each test fixture, after `create_app(...)` returns, register two test-only routes directly on the returned `FastAPI` instance:

  ```python
  # test-only; registered on the app fixture, not in production code.
  @app.get("/__test__/app-error")
  async def _raise_app_error() -> None:
      raise SomeConcreteAppError(...)   # a concrete AppError subclass from app.shared.errors

  @app.get("/__test__/unhandled")
  async def _raise_unhandled() -> None:
      raise RuntimeError("boom")
  ```

  These routes live only in the test module (they are added via `app.router.add_api_route(...)` or the decorator form on the app fixture); nothing under `app/` is modified to support them. The tests then hit `/__test__/app-error` to assert the Problem Details shape and `/__test__/unhandled` to assert the sanitized 500. The `X-Request-ID` echo test can target `/healthz` (already mounted by T-505) so it does not depend on the test-only routes.

Tests required: see above.

Acceptance criteria: tests pass against an empty domain set.

Commands: `make test`.

Common failure modes: relying on a real DB; using TestClient default headers.

Review checklist: assertions cover type/title/status/detail/code/request_id and content type.


> Five port modules. Pure Protocols + value types. No SDK imports. Each task creates `__init__.py` re-exporting only the Protocol(s) and value types listed.
