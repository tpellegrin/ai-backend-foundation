## Implementation requirements
Add async helpers in `app/auth/persistence.py`:

```python
async def mark_refresh_token_replaced(
    session: AsyncSession,
    *,
    token_id: UUID,
    replaced_by: UUID,
    revoked_at: datetime,
) -> RefreshTokenRecord | None: ...

async def revoke_refresh_token(
    session: AsyncSession,
    *,
    token_id: UUID,
    revoked_at: datetime,
) -> RefreshTokenRecord | None: ...

async def revoke_refresh_token_family(
    session: AsyncSession,
    *,
    family_id: UUID,
    revoked_at: datetime,
) -> int: ...

async def get_refresh_token_family_state(
    session: AsyncSession,
    *,
    family_id: UUID,
) -> Sequence[RefreshTokenRecord]: ...
```

- Helpers must accept `AsyncSession`.
- Helpers must accept `revoked_at` from the caller.
- Helpers should `flush()` but not `commit()`.
- Transaction ownership belongs to service/application code.
- Helpers return `RefreshTokenRecord`, `Sequence[RefreshTokenRecord]`, `int`, or `None` as appropriate.
- ORM rows must not leak.
- Service-level rotation policy remains out of scope.

## Tests required
- Rotation marks an old refresh token as replaced.
- Single-token revocation sets `revoked_at` only on that token.
- Family revocation sets `revoked_at` on all tokens in a family that need revocation.
- Family state lookup returns domain/read-model records, not rows.
- Helpers flush but do not require committing inside the helper.
- Integration tests use the existing T-905 database/Testcontainers pattern.
- Tests do not mutate `os.environ`.
