# Task T-911 — Scope-based authorization dependency

## Purpose
Add a reusable API-edge authorization dependency for requiring an authenticated, active principal with a specific scope. This enables fine-grained access control for internal resources like prompt templates.

## Depends on
- T-901

## Acceptance Criteria
- A reusable dependency factory `require_scope(scope: str)` is implemented in `app/auth/deps.py` or `app/auth/policies.py`.
- The dependency factory has the following signature: `require_scope(scope: str) -> Callable[[AuthenticatedUser], AuthenticatedUser]`.
- It relies on `get_current_user` to obtain the authenticated principal.
- If the user is missing the required scope, it raises `app.shared.errors.AppError` with a 403 Forbidden status (or a specialized `ForbiddenError` if one exists in the project).
- Missing authentication (no token) continues to result in `401 Unauthorized` via `get_current_user`.
- Inactive/disabled users continue to result in `401 Unauthorized` (or the existing behavior for disabled users).
- The dependency is used as follows: `Annotated[AuthenticatedUser, Depends(require_scope("some:scope"))]`.
- Tests are added to `app/auth/tests/test_deps.py` or `app/auth/tests/test_policies.py` covering:
    - Missing token (401).
    - Valid token with required scope (200/Success).
    - Valid token without required scope (403).
    - Token for a disabled user (Existing 401/403 behavior preserved).

## Allowed Files
- `app/auth/deps.py`
- `app/auth/policies.py`
- `app/auth/tests/test_deps.py`
- `app/auth/tests/test_policies.py`

## Forbidden Changes
- Do not change password hashing logic.
- Do not change token signing or verification logic.
- Do not change auth persistence or database schemas.
- Do not introduce roles or administrative tables.
- Do not create prompt-specific authorization code in the auth module.
- Do not return stack traces or internal secrets in error responses.

## Reference implementation hint
```python
def require_scope(scope: str):
    def dependency(user: Annotated[AuthenticatedUser, Depends(get_current_user)]) -> AuthenticatedUser:
        if scope not in user.scopes:
            raise AppError(
                code="insufficient-scope",
                title="Insufficient scope",
                status=403,
                detail=f"Required scope: {scope}",
            )
        return user
    return dependency
```
